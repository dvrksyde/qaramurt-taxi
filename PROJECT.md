# QaramurtTaxi — Полная документация проекта

## Цель проекта

Региональная система управления таксопарком для посёлка **Карамурт** и соседних сёл Южного Казахстана (Шымкент, Сайрам, Ленгер, Манкент, Аксукент, Низамабад, Коксайек, Аккала, Мадени, Саркырама, Шапырашты, Ханарык, Акбулак).

Система решает три задачи:
1. **Диспетчерский мониторинг** — операторы видят водителей на карте и создают заказы
2. **Умное распределение заказов** — автоматическая/ручная раздача заказов водителям
3. **Мобильное приложение водителя** — приём заказов, GPS-трекинг, расчёт дистанции и стоимости

## Жизненный цикл заказа

```
pending → assigned → arrived → in_progress → completed
                                           ↘ canceled
```

| Статус | Описание |
|---|---|
| `pending` | Заказ создан, ищет водителя |
| `assigned` | Водитель принял заказ |
| `arrived` | Водитель прибыл к клиенту |
| `in_progress` | Поездка началась, счётчик считает |
| `completed` | Поездка завершена, цена рассчитана |
| `canceled` | Отменён |

## Методы распределения заказов

| Метод | Описание |
|---|---|
| `automatic` | Пошаговое расширение радиуса: 1.5км → 3км → 5км → все. На каждом шаге: сортировка по уровню (золото→серебро→бронза), потом по расстоянию. Интервал 15 сек. |
| `broadcast` | Мгновенная рассылка ВСЕМ подключённым водителям без фильтров |
| `map_pick` | Диспетчер выбирает водителя на карте |
| `list_pick` | Диспетчер выбирает из списка |
| `sequential` | Поочерёдно каждому водителю |

### Система уровней водителей
- **Золото** — топ водители, получают заказы первыми
- **Серебро** — средний уровень
- **Бронза** — новые водители
- **Заблокированные** — не получают заказы

## Расчёт дистанции и стоимости

### На телефоне водителя (в реальном времени)
```
GPS → Kalman-фильтр → Отсев плохих точек → Накопление дистанции
```

**Фильтры GPS:**
- Точность > 50м → пропуск
- Скорость > 140 км/ч → телепорт, пропуск
- Дистанция < 10м → шум, пропуск
- Дистанция > 150м за шаг → пропуск
- Пауза > 30 сек → реанкоровка Kalman
- Скорость < 2 м/с + дистанция < 20м → стоянка, пропуск

### На сервере (финальный расчёт)
```
GPS-точки из БД → Haversine → OSRM map matching → Итоговая дистанция
```

Сервер берёт `max(клиентская, серверная)` дистанцию и цену.

### Формула цены
```
Итог = Подача
     + cityKm × cityRate
     + outOfCityKm × outOfCityRate
     + (outOfCitySeconds ÷ 60) × 25₸
     + waitingFee
```

Округление до ближайших 5₸.

## Типы заказов

| Тип | Описание |
|---|---|
| Обычный | assigned→arrived→in_progress→completed |
| **Бордюр** (curbside) | Создаётся прямо в `in_progress`, сессия создаётся сразу |
| Доставка | Фиксированная цена, GPS не трекается |
| Плановый | Создаётся заранее с датой/временем |

---

## Зональное ценообразование

Граница города хранится в PostGIS (`geozones.type = 'city_boundary'`).

При движении за городом:
- Км считаются по `outOfCityKmRate` (дороже)
- Дополнительно +25₸/мин за каждую минуту за городом
- Socket-событие `zone_change` уведомляет водителя о пересечении границы
- Клиентская детекция работает локально по закешированному GeoJSON-полигону (без интернета)

## Офлайн-поддержка (мобильное приложение)

| Сценарий | Решение |
|---|---|
| GPS-точки не доходят | Очередь в SecureStore, батч по 25 точек, флуш при сети |
| Смена статуса (arrived/in_progress) | `savePendingStatus` → ретрай каждые 15 сек |
| Завершение поездки | `savePendingCompletion` → перед синком флушит GPS-точки → PATCH → очистка |
| Возврат сети | Socket `connect` → мгновенный синк |
| Приложение убито Android | Метрики счётчика в SecureStore → восстановление при старте |
| Нет сессии (офлайн в момент старта) | Сессия создаётся лениво при первом успешном флуше |
